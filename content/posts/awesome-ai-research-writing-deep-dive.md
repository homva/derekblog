> **太长不看**：一个汇集了 MSRA、字节 Seed、上交 AI Lab 等顶尖机构研究员日常使用的论文写作 Prompt 模板和 Agent Skills 的开源项目。短短几个月狂揽 23,700+ stars，核心卖点是「复制即用」——别再浪费时间在 prompt 调试上了。

## 项目概况

[**awesome-ai-research-writing**](https://github.com/Leey21/awesome-ai-research-writing) 是一个面向科研写作的开源 Prompt 模板库，由 [Leey21](https://github.com/Leey21) 创建，创建于 2026 年 1 月，目前数据如下：

- ⭐ **Stars**: 23,715
- 🍴 **Forks**: 1,910
- 📦 **README 大小**: 54KB（内容量惊人）
- 📅 **最后更新**: 2026 年 3 月

项目的核心动机直击痛点：**在学术圈，prompt 工程正在成为一种"隐性资源"——顶尖研究组有完善的模板库，而大多数人还在从零摸索。** 他们调研了微软亚洲研究院（MSRA）、字节跳动 Seed、上海人工智能实验室等机构的研究员，以及北大、中科大、上交的硕博同学，将这些一线人员日常使用的写作技巧全部开源。

## 项目结构

项目内容分为两大部分：

**Part I — 写作 Prompt 集合**：15+ 个场景的高质量 Prompt 模板

**Part II — 论文写作相关的 Agent Skills**：Skills 配置教程与核心 Skills 抽取

此外还衍生了一个独立项目：[**arxiv-translator**](https://github.com/Leey21/arxiv-translator)，支持从 LaTeX 源码直接翻译并编译 PDF。

## Part I：Prompt 模板全景拆解

### 1. 中转英 🇨🇳→🇬🇧

面向 LaTeX 场景的中译英 Prompt。几个亮点设计：

- **角色设定**：兼具"顶尖科研写作专家"和"资深 ICML/ICLR 审稿人"双重身份
- **约束严格**：禁用加粗/斜体/引号、禁用破折号、拒绝 `\item` 列表、统一使用一般现在时
- **LaTeX 友好**：自动转义 `%`、`_`、`&` 等特殊字符，保留数学公式
- **输出规范**：Part 1 输出纯 LaTeX 英文，Part 2 输出中文直译用于核对
- **自查机制**：内置 Execution Protocol，要求 AI 在输出前以审稿人视角自我审查

### 2. 英转中 🇬🇧→🇨🇳

反向翻译，专门针对"快速理解英文论文"的场景：

- **语法清洗**：自动删除 `\cite{}`、`\ref{}`、`\label{}` 等干扰阅读的索引命令
- **公式自然化**：`$\alpha$` → alpha，`\frac{a}{b}` → a/b，不保留 LaTeX 语法
- **直译原则**：严格对应原文，不做润色和逻辑优化，保留原文的生硬表达

这个设计很聪明——翻译论文不是为了发表，而是为了读懂，直译比润色更合适。

### 3. 中转中 📝

面向 Word 场景的中文学术重写：

- 将碎片化、口语化的草稿重写为符合《计算机学报》《软件学报》标准的学术正文
- 逻辑重组（不逐句润色，而是识别主线后重构）
- 一个段落一个核心观点
- Word 友好：输出纯文本，禁用 Markdown 格式

### 4. 缩写 & 5. 扩写 ✂️📏

这两个是成对的微调工具，设计理念一致：**小幅度、保信息、不注水**。

- 缩写：减少约 5-15 个单词，通过句法压缩和剔除冗余填充词
- 扩写：增加约 5-15 个单词，挖掘隐含结论、增强逻辑连接
- 都要求保持 LaTeX 纯净，禁止大删大改

这个"微调"思路很实用——不是重写，而是精细打磨。

### 6. 表达润色（英文论文）✨

对标 NeurIPS/ICLR/ICML 顶会标准的英文润色：

- **零错误原则**：拼写、语法、标点、冠词全修正
- **禁用缩写**：it's → it is，doesn't → does not
- **避免所有格**：METHOD's performance → the performance of METHOD
- **禁止展开缩写**：LLM 保持 LLM，不展开为 Large Language Models
- 保留 `\cite{}`、`\ref{}` 等 LaTeX 命令

### 7. 表达润色（中文论文）📄

面向中文核心期刊的润色，核心理念是**克制**：

- 仅在检测到口语化、语法错误、逻辑断层时才修改
- 如果原文已经清晰准确，**保留原样不动**
- 禁止为了追求形式变化而强行替换同义词
- 拒绝陈旧公文腔（不无故将"旨在"改"拟"，"是"改"系"）

> 这个"宁缺毋滥"的设计哲学值得所有 AI 写作工具学习。

### 8. 逻辑检查 🔍

论文终稿的"红线审查"：

- **仅报错原则**：只有致命逻辑错误才提意见，风格问题直接忽略
- 审查维度：致命逻辑、术语一致性、严重语病
- 高容忍度：预设草稿已经过多轮修改，质量较高

这个 Prompt 的自我定位非常清晰——不是来刷存在感的，是来做最后一道安全网的。

### 9. 去 AI 味（LaTeX 英文 + Word 中文）🤖→👤

这是整个项目**最有价值的部分之一**。

项目给出了一份 **AI 味词汇黑名单**（40+ 个词）：

> leverage, delve into, tapestry, underscore, unveil, pivotal, nuance, nuanced, testament, transcendent, galvanize, ameliorate, elucidate, scrutinize...

这些都是 AI 生成文本中过度使用的"高级词汇"。Prompt 的策略是：

- 用朴实精准的词替代（use 替代 leverage，investigate 替代 delve into）
- 移除机械连接词（First and foremost, It is worth noting that）
- 减少破折号使用，改用逗号或从句
- **宁缺毋滥**：如果原文已经自然地道，不修改

中文版还特别针对"翻译腔"做了优化：

- 拆分英式长定语结构（"一个...的...的..."）
- 限制"被"字句，改用主动语态
- 消除空泛的情感渲染（"毋庸置疑"、"范式转移"、"颠覆性"）

### 10. 论文架构图 🎨

面向 CVPR/NeurIPS 级别的学术架构图生成 Prompt：

- 扁平化矢量风格，DeepMind/OpenAI 论文美学
- 淡色系/柔和色调，纯白背景
- 全英文标注
- 禁止卡通感、油画感、3D 阴影

还提供了一个针对 Nano Banana 的英文版本，反馈效果更好。

### 11. 实验绘图推荐 📊

从 19 种标准学术图表中选择最佳方案：

- 数值对比（分组柱状图、帕累托前沿图、雷达图）
- 趋势收敛（带置信区域的折线图、局部放大图）
- 模型评估（ROC、PR 曲线）
- 关系矩阵（热力图、散点图、气泡图）
- 统计分布（小提琴图、箱线图）
- 复合布局（双 Y 轴、柱折组合、分面网格）

还附带尺度适应性建议：断裂坐标轴、对数坐标、归一化处理。

### 12. 图/表标题生成 🏷️

自动将中文描述转为符合顶会规范的英文图标题和表标题：

- 名词短语 → Title Case，末尾无句号
- 完整句子 → Sentence case，末尾加句号
- 极简原则：去除 "The figure shows" 这类冗余开头
- 表格推荐用 Comparison with、Ablation study on、Results on 等标准表达

### 13. 实验分析 📈

将实验数据转为 LaTeX 分析段落：

- 拒绝简单报账式描述（不"说 A 是 0.5，B 是 0.6"）
- 重点关注 SOTA 比较、参数敏感性、性能与效率权衡、消融实验
- 强制使用 `\paragraph{核心结论}` + 分析文本的格式
- 严禁编造数据或夸大提升幅度

### 14. Reviewer 视角审视 👀

模拟审稿人对论文进行全面评估：

- 区分"真正致命的问题"和"可以在修订期内解决的小问题"
- 审查维度：社区贡献、严谨性、一致性
- 评分忠实反映论文实际水平
- 省略客套表述，直击核心判断

### 15. 模型选择 🤔

AI 辅助推荐最适合当前任务的模型。

## Part II：Agent Skills

项目除了 Prompt 模板，还提供了 Agent Skills 的配置教程和核心 Skills 抽取。Agent Skills 作为新兴技术，能更强大、更自动化地助力论文写作，但存在一定使用门槛。项目提供：

- Skills 配置指南
- 写作相关的核心 Skills 抽取
- 使用场景与示例 Prompt

这部分降低了 Agent Skills 的使用门槛，让用户不需要从零配置就能上手。

## 核心设计理念总结

通读全部 Prompt 后，我发现这个项目有几个贯穿始终的设计哲学：

### 1. 角色设定精准

每个 Prompt 都给了 AI 一个具体的、有权威感的角色（"资深学术编辑"、"顶级会议审稿人"、"数据可视化专家"），而不是泛泛的"你是一个助手"。这种角色锚定让 AI 的输出质量有质的提升。

### 2. 约束 > 指令

几乎所有 Prompt 都用了大量的 Constraints 来限制 AI 的行为，而不是单纯告诉它"要做什么"。通过定义"不要做什么"来约束输出，比单纯说"要做好"有效得多。

### 3. 宁缺毋滥的自我克制

润色、去 AI 味等场景中，反复强调"如果原文已经够好，不要修改"。这是非常成熟的设计——AI 不应该为了刷存在感而制造无意义的改动。

### 4. 输出格式标准化

几乎所有 Prompt 都规定了 Part 1 / Part 2 / Part 3 的分块输出，确保结果可核对、可追溯、无多余废话。

### 5. LaTeX/Word 场景分离

同一个任务（如润色），针对 LaTeX 和 Word 场景分别设计 Prompt，体现了对实际工作流的深刻理解。

## 个人评价

### ✅ 优点

- **实战打磨**：不是实验室里凭空写的玩具，而是一线科研人员真实使用的模板
- **场景覆盖全面**：从翻译、润色、检查、出图到投稿评审，覆盖了论文写作的全生命周期
- **设计精良**：每个 Prompt 的约束条件写得很细，输出格式明确，自带自查机制
- **开源精神**：把"隐性资源"开放出来，对没有顶尖团队背景的科研人员来说是真正的福利
- **数据漂亮**：2.3万+ stars 说明社区认可度极高

### ⚠️ 局限

- **本质是 Prompt 集合**：技术深度有限，没有构建独立的工具链或平台
- **缺少版本管理**：README 长达 54KB，所有内容塞在一个文件里，查找特定 Prompt 不够方便
- **没有自动化集成**：需要手动复制粘贴到聊天框，如果能做成 CLI 工具或 IDE 插件会更好
- **无开源许可证**：仓库未声明 License，使用和二次分发存在法律不确定性

### 🔮 对我们的启发

作为一个技术团队 Leader，我从这个项目中学到了几点可以复用到工作中的东西：

1. **Prompt 设计的方法论值得借鉴**：角色设定 + 严格约束 + 输出格式 + 自查机制，这个模式可以复用到我们日常的技术文档生成、代码审查、需求分析等场景
2. **"去 AI 味"的思路可以用在非科研场景**：日常公文、技术报告、产品文档同样面临 AI 味问题
3. **场景分离的理念**：同一个任务，不同输出目标（LaTeX vs Word）需要不同的 Prompt 策略——这一点在 API 文档 vs 用户手册的生成中同样适用
4. **知识库的"隐性资源显性化"**：我们团队内部也有很多沉淀下来的经验没有文档化，这个项目给了一个很好的参考

## 结语

`awesome-ai-research-writing` 不是革命性的技术突破，但它把一件小事做到了极致——**让 AI 辅助科研写作真正可用、好用**。

它没有造新轮子，而是把顶尖实验室里经过实战检验的最佳实践开源出来，让所有人都能直接受益。这种务实的开源精神，值得所有做工具链的人学习。

如果你在写论文，这个项目值得收藏。即使你不做科研，里面的 Prompt 设计思路，也值得每个和 AI 打交道的人学习。
